# Launch checklist

The build is dual-mode. With empty keys it runs as a demo. With Supabase keys
it is live: real accounts, real persistence. Setup time to live sign-ups: 60
to 90 minutes.

## Go live

1. Push to GitHub and import the repo into Vercel (framework preset: Other).
   Netlify or GitHub Pages also work. There is no build step at deploy time;
   HTML is generated by the Python builders and committed.
2. Database. Existing production: follow supabase/migrations/README.md (one
   time baseline pull, then `supabase db push`). Fresh environment: run
   supabase/schema.sql, then schema_rbac.sql, then schema_keystone.sql in the
   SQL editor, then adopt the migration baseline the same way.
3. Edge functions: `supabase functions deploy checkout` and
   `supabase functions deploy checkout-webhook`. The claim gate lives in the
   checkout function; the site is not enforcing v4.0 policy until both the
   migrations and these deploys land.
4. Supabase, Authentication, URL Configuration: set Site URL to the deployed
   domain and add it to Redirect URLs. Auth is email and password only. The
   Confirm email toggle decides whether new accounts must confirm before
   first sign-in; the client handles either setting.
5. Paste the project URL and anon key into assets/js/config.js. Redeploy.
6. Test the participant loop: create an account on /login.html, take the
   Profile, land on the dashboard. Test the course loop: grant yourself
   circle_leader in Admin, claim your own email on the Facilitator Desk,
   then enroll from /enroll.html at zero dollars.

## Optional, same day

7. Stripe: for the membership Payment Link, paste its URL into config.js.
   Webhooks: add the endpoint for checkout.session.completed, set
   STRIPE_WEBHOOK_SECRET, deploy with --no-verify-jwt. Signatures are
   verified in code; the functions refuse unsigned events.
8. Resend: verify the domain, set RESEND_API_KEY, deploy send-email.
   Templates are in emails/. Supabase's built-in sender is capped at two
   emails per hour; production sign-up email requires Resend before any
   launch push.
9. Video: paste Vimeo links per course video in the Admin course console.
   Watch tracking measures against the entered lengths.

## The Keystone Father Profile

The platform ships the complete 128-item, 26-scale instrument, scored
against published scale norms with two completion modes (all at once, or
section by section with resume). The instrument data lives in
assets/js/keystone-data.js. The open norms question (9,232 versus 2,066) is
logged in POSITIONING.md section 8 and waits on Dr. Ken Canfield; change
neither figure until he rules.

## Roles and dashboards

Grant roles in Admin, People and roles. Keys: member, circle_leader,
org_admin, content_reviewer, instructor, admin. Display names and the full
permission map are in ROLES.md and FC-REQ-001.
