# Fathers.com Platform

The assessment and course platform of the National Center for Fathering.
Static front-end generated by Python builders, Supabase backend, Vercel
hosting. v4.0 model: NCF certifies organizations and facilitators;
participants pay nothing, ever. Decisions and their rationale live in
POSITIONING.md; the full requirements document is FC-REQ-001.

## Run locally

```
python3 -m http.server 8000
```

Open http://localhost:8000. With empty keys in `assets/js/config.js` the site
runs in demo mode; with keys it is live against Supabase.

## Build

All HTML is generated. Never hand-edit a page; edit the builder and
regenerate. CI fails any push where regeneration produces a diff.

```
python3 build_pages.py        # public and participant pages
python3 build_dashboards.py   # admin, org, facilitator, studio, snapshot
python3 build_emails.py       # Resend templates in emails/
```

## Test

```
pip install -r requirements-dev.txt
playwright install chromium
pytest tests/e2e
```

The suite serves the built site locally with all external requests aborted:
offline, deterministic, and unable to touch production.

## Security model in one paragraph

Row-level security is the boundary; the browser is untrusted. The committed
Supabase key is the public anon role. Secrets exist only as edge-function
secrets. Enrollment eligibility (the participant claim) and price are decided
in the checkout edge function, never in the client. Stripe webhooks verify
signatures and refuse unsigned events. Public verification of certificates,
facilitators, and organizations goes through limited database views that
expose status, including revoked.
